Infrastructure

The Data is being handled by a MySQL Database. The Data is being extracted by a custom API written in python using the Flask Lib The Frontend section is a HTML5 Application using the following Libs:
  • Bootstrap
  • Bootstrap-slider
  • Morrisjs
  • D3

User Manual

Welcome to the user guide for the WePredict Health care Visualization project This document will describe how to use the site and how to get the site working on your own servers. The Project can be accessible view here: http://wepredict.robrotheram.com/ The Site is separated in to 2 separate parts. The back-end section that is a Rest API which serves the data from the database. The second part is the visualization web application that can live apart from the backed and consumes the API from the back end and generate vitalizations from the data Full Documentation for the project is available here: http://robrotheram.github.io/Documentation/ The project is designed to be very flexible with the setup this means that there are multiple combinations can exits. For example the front end and the Back-end of the application can live on different servers. Or alternatively the Backed can be re-coded into a different language and so long as the URL paths remain the same only a minor change to the front end will be needed.

2 CCG Page

 The CCG page will be the main page that the user will spend on. It centers on one main visualization that will help users compare the CCG practices to see where a problem may occur. The page also contains other visualization such as overview graphs at the bottom of the page. The page also contains a select option to drill down onto a single practice.
  1. Menu Menu
  2. Select practice
  3. Name of the practice

2.1 Main visualization

 The main part of the CCG page is the customizable chart section. In this visualization you can set the chart type and data type from the drop down box below. Other features of this visualization is that you can click on the hide all button which will hide all the data series from the graph. You can now click on individual series from the legend so that you can compare several practices together.
  1. Graph view of the visualization
  2. Legend of the visualization, Clicking on each item in the Legend it will show or hide that field in the visualization
  3. button to hide all fields from the visualization
  4. button to Show all fields from the visualization
  5. Select which data attribute to visualize
  6. Select what type of chart of the visualize

2.2 Other CCG charts

 On the CCG page is a number of overview charts these rest at the bottom of the CCG page. They contain information about the average data for all of the CCG the container that holds theses panel can horizontally scroll reviling different data attributes.

The other visualization on the CCG page is a small comparison visualization. it leverages a scrolling table to show each practice and the data. It allows for a quick overview glance to spot patterns or notice some anomaly in the data and then use the practice page to look at that practice further.
